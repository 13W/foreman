# t3.5 — Final Proxy Integration Design

**Date:** 2026-04-24  
**Scope:** `packages/proxy` + `packages/shared` interface updates  
**Goal:** Replace the `ProxyServer` stub with a full orchestrator wiring together all previously built components; implement `DefaultA2AServer` as the HTTP facade.

## 1. Shared Interface Updates

### 1.0 NEW: `packages/shared/src/permissions.ts`

`PermissionDecision` is used by both A2A (`requestInput` return) and ACP (`PromptEvent.respond`). `PermissionRequestType` is used in `PermissionRequest` (a2a) and ACP permission mapping. Both live here to avoid a2a↔acp circular imports.

```typescript
export type PermissionDecision =
  | { kind: 'allow_once' }
  | { kind: 'allow_always' }
  | { kind: 'reject_once' }
  | { kind: 'reject_always' }
  | { kind: 'cancelled' }; // reserved for session/cancel propagation only

// Domain-agnostic name for the three ACP permission categories.
// ACP-specific field names ('fs.read' etc.) are the same strings but typed here
// to avoid importing from acp/types.ts into a2a/types.ts.
export type PermissionRequestType = 'fs.read' | 'fs.write' | 'terminal.create';
```

Both `a2a/types.ts` and `acp/types.ts` import from `../permissions.js`. Re-export both from the shared barrel `index.ts`.

### 1.1 `packages/shared/src/a2a/types.ts`

**Replace `PermissionRequest`** — remove `requestId` (internal to executor), remove generic `options`, add typed fields. Use `PermissionRequestType` from `permissions.ts` (not `ACPPermissionType` from `acp/types.ts` — that would re-introduce the circular import):

```typescript
import type { PermissionDecision, PermissionRequestType } from '../permissions.js';
export type { PermissionDecision, PermissionRequestType };

export interface PermissionRequest {
  type: PermissionRequestType;
  path?: string;    // present for fs.read / fs.write
  command?: string; // present for terminal.create
  message: string;  // human-readable summary for the operator
}
```

**Remove `InputDecision`** — no longer needed; superseded by `PermissionDecision`.

**Update `AgentCardMetadata`** — remove `capabilities`, add typed skills:

```typescript
import type { AgentSkill } from '@a2a-js/sdk';
export type { AgentSkill };

export interface AgentCardMetadata {
  name: string;
  url: string;
  version: string;
  description?: string;
  skills?: AgentSkill[];
}
```

### 1.2 `packages/shared/src/a2a/server.ts`

Update `requestInput` return type:

```typescript
requestInput(
  taskId: string,
  request: PermissionRequest,
  options?: { timeoutMs?: number },

```

### 1.3 `packages/shared/src/acp/types.ts`

**Add `PromptEvent`** discriminated union:

```typescript
import type { PermissionDecision } from '../permissions.js';

export type PromptEvent =
  | { kind: 'agent_message_chunk'; content: ContentBlock }
  | { kind: 'tool_call'; update: ToolCallUpdate }
  | { kind: 'tool_call_update'; update: ToolCallUpdate }

      kind: 'permission_request';
```

`PermissionDecision` imported from `../permissions.js` (not from `../a2a/types.js` — avoids circular import).

**Remove `PromptResult`** — no longer exported.

### 1.4 `packages/shared/src/acp/client.ts`

## 2. `ProxyAgentExecutor` (`packages/proxy/src/a2a/executor.ts`)

Implements `AgentExecutor` from `@a2a-js/sdk`. Coordinates task lifetime, streaming, and permission escalation. Testable without HTTP.

### 2.1 Data structures

```typescript
interface Deferred<T> {
  promise: Promise<T>;
  resolve: (value: T) => void;
  reject: (reason: unknown) => void;
}

interface TaskContext {
  /**
   * Current bus for this task. Updated on second execute() call (input round-trip):
   * the SDK creates a fresh bus for each execute() invocation, so after requestInput
   * publishes input-required, the next execute() call provides a new bus reference here.
   */
  bus: ExecutionEventBus;
  contextId: string;
  /**
   * Resolved ONLY from completeTask() or cancelTask() — never from the bus or runPrompt.
   * The ProxyServer awaits this to know the task reached a terminal state.
   */
  completionDeferred: Deferred<void>;
  /**
   * Present while awaiting human input. ACP blocks the subprocess during this period,
   * so only one pendingInput can be active per task at any time.
   */
  pendingInput?: { id: string; deferred: Deferred<PermissionDecision> };
  /** Called by cancelTask() to forward cancellation into the ACP session. */
  cancelFn?: () => void;
}
```

### 2.2 `execute(ctx, bus)`

```
1. Look up existing TaskContext for ctx.task.id
2. If found (second execute — input round-trip):
   a. Update ctx.bus to new bus
   b. Parse PermissionDecision from ctx.message using parsePermissionDecision()
   c. Resolve pendingInput.deferred with parsed decision
   d. Return (ProxyServer's runPrompt loop resumes and calls respond())
3. If not found (first execute):
   a. Create TaskContext with completionDeferred, store in tasks map
   b. Publish working status: { state: 'working' }
   c. Parse TaskPayload from ctx.message using parseTaskPayload()
      - On failure: completeTask with buildErrorTaskResult(err)
   d. Call taskHandler(payload, { taskId, agentUrl: this.agentUrl })
      - taskHandler is fire-and-forget from execute's perspective
      - On rejection: log.error + completeTask with buildErrorTaskResult(err)
   e. Return (taskHandler runs independently, resolves completionDeferred when done)
```

### 2.3 `cancelTask(taskId, bus)`

```
1. Look up TaskContext
2. If not found: publish cancelled status on bus; return
3. Call ctx.cancelFn() if set
4. If pendingInput present: resolve its deferred with { kind: 'cancelled' }
5. Resolve completionDeferred (cancellation is terminal)
6. Remove from tasks map
```

### 2.4 `requestInput(taskId, request, opts)`

`opts.timeoutMs` is always provided by the caller (`ProxyServer` reads `config.permissions.permission_timeout_sec` and converts to ms). Executor has no config dependency.

```
1. Look up TaskContext; throw if not found
2. Generate requestId = randomUUID()
3. Create Deferred<PermissionDecision>
4. Set ctx.pendingInput = { id: requestId, deferred }
5. timeoutMs = opts.timeoutMs  (required — caller must supply)
6. Set timeout: on expiry, reject deferred with PermissionTimeoutError
   TODO: store timer handle in TaskContext and clearTimeout on normal completion
7. Publish on ctx.bus: status-update { state: 'input-required', final: true, metadata: { requestId, request } }
   NOTE: final:true ends the event queue without bus.finished() — correct per SDK mechanics
8. Await deferred.promise
9. Clear timeout, clear ctx.pendingInput
10. Return resolved PermissionDecision
```

### 2.5 Other methods

- `sendUpdate(taskId, event)` — look up ctx, publish event on `ctx.bus`
- `completeTask(taskId, result)` — publish terminal status, resolve `completionDeferred`, remove from map
- `setCancelFn(taskId, fn)` — set `ctx.cancelFn`

## 3. `DefaultA2AServer` (`packages/proxy/src/a2a/server.ts`)

Thin HTTP facade implementing `A2AServer`. Owns the Express app, routes, and `ProxyAgentExecutor`.

### 3.1 Structure

```typescript
export class DefaultA2AServer implements A2AServer {
  private readonly executor: ProxyAgentExecutor;
  private readonly app: express.Application;
  private server?: http.Server;
}
```

### 3.2 `register(agentCard)`

```
1. sdkCard = buildSdkAgentCard(agentCard)
2. Construct ProxyAgentExecutor(taskHandler, agentCard.url)
3. Construct DefaultRequestHandler(executor, new InMemoryTaskStore())
4. Mount routes:
   app.use(`/${AGENT_CARD_PATH}`, agentCardHandler({ agentCardProvider: requestHandler }))
   app.post('/a2a/jsonrpc', jsonRpcHandler(requestHandler))
```

`AGENT_CARD_PATH` is `'.well-known/agent-card.json'` (without leading `/`) from `@a2a-js/sdk`. The `agentCard.url` is passed as-is; ProxyServer sets it to the correct public URL before calling `register()`.

**`buildSdkAgentCard(metadata: AgentCardMetadata): SDK.AgentCard`** — adapts our slim type to the full SDK schema. Lives in `DefaultA2AServer` (private static) or `mappers.ts`:

```typescript
{
  ...metadata,
  protocolVersion: '0.3.0',
  capabilities: { streaming: true, pushNotifications: false },
  defaultInputModes: ['text'],
  defaultOutputModes: ['text'],
}
```

### 3.3 `sendUpdate / completeTask / requestInput`

All delegate to `this.executor`.

### 3.4 `listen(bindAddr)`

```
1. Parse bindAddr as "host:port" (IPv6: "[::1]:port")
2. Create http.Server from Express app
3. server.listen(port, host)
4. Store server handle for close()
```

### 3.5 `close()`

Calls `server.close()` — not part of `A2AServer` interface, used by `ProxyServer.shutdown()`.

## 4. `ProxyServer` (`packages/proxy/src/proxy-server.ts`)

Full orchestrator. Wires together all components.

### 4.1 Constructor

```typescript
constructor(
  private readonly config: ProxyConfig,
  private readonly a2aServer: A2AServer,
  private readonly subprocessPool: SubprocessPool,
  private readonly worktreeManager: WorktreeManager,
  private readonly acpClientManager: ACPClientManager, // used only for cancelSession in setCancelFn
)
```

Default construction (CLI path) instantiates concrete classes. DI constructor used in tests.
`acpClientManager` is needed only to call `cancelSession()` inside the `setCancelFn` closure — `SubprocessPool` owns session creation.

### 4.2 `start()`

```
1. validatePersonalMcps(config.mcps.personal) — throws McpNameCollisionError on collision
2. Build AgentCardMetadata:
   - url: `http://${config.proxy.bind}` (loopback-only per BindSchema)
   - name, version, description from config.proxy
   - skills from config.role.skills (mapped McpServerSpec → AgentSkill)
3. a2aServer.register(agentCard)
4. a2aServer.onTask(handleTask)
5. a2aServer.listen(config.proxy.bind)
```

### 4.3 `handleTask(payload, handle)`

`SubprocessPool.acquireSession(cwd, mcpServers)` handles both subprocess selection and session creation in one call. It returns `PooledSession { subprocess, session, release() }`.

```
1. baseBranch = payload.base_branch ?? config.worktrees.default_base_branch
   - If baseBranch is empty string after fallback: fail with MissingBaseBranchError
     (proxy-spec §8.1: base branch must be non-empty)
2. worktreeResult = await worktreeManager.createForTask(handle.taskId, baseBranch)
   - On error: await completeTask(buildErrorTaskResult(err)); return
3. mergedMcps = mergeMcps(config.mcps.personal, payload.injected_mcps)
   - On McpNameCollisionError:
       await worktreeManager.cleanup(handle.taskId)
       await completeTask(buildErrorTaskResult(err)); return
4. pooled = await subprocessPool.acquireSession(worktreeResult.worktreePath, mergedMcps)
   - On PoolExhaustedError:
       await worktreeManager.cleanup(handle.taskId)
       await completeTask(buildErrorTaskResult(err)); return
5. a2aServer.setCancelFn(handle.taskId, () => acpClientManager.cancelSession(pooled.session))
   NOTE: must be called before runPrompt so SDK cancelTask can cascade
6. try:
     stopReason = await runPrompt(payload, handle, pooled.session, worktreeResult)
     result = buildTaskResult(stopReason, worktreeResult)
   catch err:
     result = buildErrorTaskResult(err, worktreeResult)
   finally:
     await pooled.release()  // decrements ref-count, disposes subprocess when hits 0
     await worktreeManager.cleanup(handle.taskId, result.status === 'completed' ? 'completed' : 'failed')
     TODO: delete session transcript temp file
7. a2aServer.completeTask(handle.taskId, result)
```

### 4.4 `runPrompt(payload, handle, session, worktreeResult)`

```
1. systemPrompt = buildSystemPrompt(config, payload)
2. stream = acpClientManager.sendPrompt(session, systemPrompt)
3. for await event of stream:
   a. if event.kind === 'permission_request':
        await handlePermissionEvent(handle.taskId, event, worktreeResult)
   b. elif event.kind === 'stop':
        return event.reason
   c. else:
        mapped = mapPromptEventToStreamEvent(event)
        if mapped != null: a2aServer.sendUpdate(handle.taskId, mapped)
```

### 4.5 `handlePermissionEvent(taskId, event, worktreeResult)`

```
1. policyDecision = evaluateProxyPolicy(event.request, worktreeResult.worktreePath)
2. if policyDecision === 'approve':
     await event.respond({ kind: 'allow_once' })
     // mapDecisionToAcpResponse handles optionId lookup from event.request.options
     return
3. timeoutMs = config.permissions.permission_timeout_sec * 1000
4. decision = await a2aServer.requestInput(taskId, mapToPermissionRequest(event), { timeoutMs })
5. await event.respond(decision)
```

`evaluateProxyPolicy` combines `evaluateFsPermission` (fs.read/fs.write) and `evaluateTerminalPermission` (terminal.create) from `permission-policy.ts`.

### 4.6 `shutdown()`

```
await subprocessPool.shutdown()
await a2aServer.close()
```

## 5. Mapping & Helper Functions (`packages/proxy/src/a2a/mappers.ts`)

### 5.1 `buildSystemPrompt(config, payload) → ContentBlock[]`

Single `TextPart`. Sections present only when non-empty/non-null:

```
=== Role ===
{config.role.description}

=== Originator Intent ===
{payload.originator_intent}

=== Task ===
{payload.description}

=== Expected Output ===          (omit if null)
{payload.expected_output}

=== Relevant Files ===           (omit if empty)
{payload.inputs.relevant_files.join('\n')}

=== Constraints ===              (omit if empty)
{payload.inputs.constraints.join('\n')}

=== Prior Task Context ===       (omit if empty)
{payload.inputs.context_from_prior_tasks.map(c => `[${c.task_id}] ${c.summary}`).join('\n')}
```

### 5.2 `mapPromptEventToStreamEvent(event) → StreamEvent | null`

| `event.kind` | `StreamEvent` | Notes |
|---|---|---|
| `agent_message_chunk` | `{ type: 'message', data: { content: event.content } }` | Verbatim |
| `tool_call` | `{ type: 'status', data: { state: 'working', message: event.update.title ?? event.update.id } }` | |
| `tool_call_update` | `{ type: 'status', data: { state: 'working', message: event.update.title ?? event.update.id } }` | |
| `plan` | `{ type: 'status', data: { state: 'working', message: \`plan updated (${event.entries.length} entries)\` } }` | Full plan forwarding is a future enhancement |
| `stop` | `null` | Handled by `runPrompt` caller |
| `permission_request` | `null` | Handled by `handlePermissionEvent` |

### 5.3 `buildTaskResult(stopReason, worktreeResult) → TaskResult`

| `stopReason` | `status` | `stop_reason` | Notes |
|---|---|---|---|
| `end_turn` | `completed` | `end_turn` | |
| `max_tokens` | `failed` | `max_tokens` | Agent response truncated |
| `refusal` | `failed` | `refusal` | |
| `cancelled` | `cancelled` | `cancelled` | |
| `timeout` (proxy-synthesized) | `failed` | `timeout` | |

Always includes `branch_ref: worktreeResult.branchName`, `summary: ''` (TODO: extract from transcript), `session_transcript_ref: ''` (TODO).

### 5.4 `buildErrorTaskResult(err, worktreeResult?) → TaskResult`

| Error type | `status` | `stop_reason` | `error.code` |
|---|---|---|---|
| `MissingBaseBranchError` | `failed` | `null` | `missing_base_branch` |
| `BaseBranchNotFoundError` | `failed` | `null` | `base_branch_not_found` |
| `McpNameCollisionError` | `failed` | `null` | `mcp_name_collision` |
| `PoolExhaustedError` | `failed` | `null` | `proxy_busy` |
| `PermissionTimeoutError` | `failed` | `timeout` | `permission_timeout` |
| anything else | `failed` | `subprocess_crash` | `internal_error` |

`error.message` always set to `err.message`. `worktreeResult` optional (may not exist for pre-flight errors).

### 5.5 `parsePermissionDecision(message: Message) → PermissionDecision`

1. Try `DataPart` where `data.kind` is present → parse with `PermissionDecisionSchema`
2. Try `TextPart` JSON → parse with `PermissionDecisionSchema`
3. On failure → `{ kind: 'reject_once' }` + `log.warn('invalid permission decision response')`

`PermissionDecisionSchema` (Zod): discriminated union on `kind`, validates against the 5 string literals.

### 5.6 `parseTaskPayload(message: Message) → TaskPayload`

1. Try `DataPart` (canonical foreman format)
2. Fallback: `TextPart` JSON
3. On failure → throw `InvalidPayloadError`

### 5.7 `mapDecisionToAcpResponse(decision, options) → RequestPermissionResponse`

```
if decision.kind === 'cancelled':
  return { outcome: 'cancelled' }

option = options.find(o => o.kind === decision.kind)
if !option:
  log.warn('agent did not provide option for decision kind', { decision, options })
  return { outcome: 'cancelled' }  // defensive fallback

return { outcome: 'selected', optionId: option.optionId }
```
